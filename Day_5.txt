What is SELinux ?
SELinux (Security-Enhanced Linux) is a Linux kernel security module that provides mandatory access control (MAC). 
It enforces strict rules on how processes, users, and files interact, reducing the impact of misconfigurations or compromised applications.


In Simple way : 

SELinux adds an extra security layer by defining:

What actions a process can perform
Which files a process can access
Which network ports/services it can interact with
It uses policies to enforce this. Even root is restricted.

Example:
Apache web server normally reads website files.
Without SELinux, if Apache is hacked, it could read other files on your server.
With SELinux, Apache cannot read anything outside its allowed folder.

Policies : define what is allowed.

SELinux is widely used in production environments where security is critical

1. Enterprise Linux servers : RHEL, CentOS, AlmaLinux, and Fedora ship with SELinux enabled by default.

2. Web hosting : Prevents compromised services (e.g., Apache, Nginx) from accessing sensitive files.

3. Databases   : SELinux restricts MySQL/PostgreSQL to specific directories and ports.

4. Container platforms
	Podman uses SELinux labels for container isolation
	OpenShift (Red Hat's Kubernetes) relies heavily on SELinux to isolate pods
5. Security-critical environments : 
	Government, finance, healthcare, and telecommunication systems.
